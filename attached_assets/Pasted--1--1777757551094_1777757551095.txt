В SPECIES_LABELS:

В SPECIES_IMAGES — используй любую существующую заглушку как placeholder.
2. Множественный выбор птиц в запросе на присмотр
В app/new-request.tsx замени одиночный выбор птицы на мультиселект. Вместо одного bird_id — массив bird_ids: string[]. Показывай чекбоксы по списку птиц пользователя. Минимум одна птица должна быть выбрана для отправки запроса.
Обнови тип SitRequest:
tsbird_ids: string[]; // вместо bird_id: string
Обнови AppContext.tsx — метод addSitRequest принимает bird_ids.

tstype SitType = "full" | "guest" | "medical_only";

В тип User добавь поле:
tssitTypes?: SitType[]; // что готов делать

tssitType: SitType; // что нужно хозяину
На экране профиля app/(tabs)/profile.tsx — добавь секцию «Что я готов делать» с мультиселектом из SIT_TYPE_LABELS. Показывай сразу под статусом готовности помогать.
На карточке соседа app/neighbor/[id].tsx — показывай иконки типов передержки которые он готов делать.
На экране создания запроса app/new-request.tsx — добавь обязательный выбор типа передержки через радиокнопки из SIT_TYPE_LABELS. Это первый вопрос после выбора птиц.
На экране поиска соседей — фильтруй по совместимости: если хозяин ищет medical_only, показывай только тех у кого есть medical_only в sitTypes.
